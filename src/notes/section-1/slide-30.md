## 2. Технический фон

### Как Memory устроена технически
1. Пользователь общается с моделью в каком-то чате
2. Продукт (а не модель) автоматически выделяет «значимые факты» из переписки — обычно через отдельный вспомогательный запрос к той же или меньшей модели
3. Эти факты сохраняются в **базе данных продукта**, привязанной к аккаунту пользователя
4. Когда пользователь открывает новый чат, продукт **автоматически добавляет релевантные заметки в системный промт** этого нового чата
5. Модель видит заметки как обычный текст системных инструкций

### Ключевое архитектурное разделение
- **Модель** — обученные веса, неизменяемые после релиза
- **Продукт** — приложение поверх модели: хранилище истории, Memory, поиск, рендеринг, биллинг

Memory полностью на стороне **продукта**. Модель ничего не «запоминает». Это разделение проходит через всю Часть II.

### Где Memory есть
- **ChatGPT** — собственно «Memory» (включено по умолчанию в Plus)
- **Claude.ai** — функционал был добавлен в 2025 в виде «Projects» и «Memory»
- **Gemini** — «context personalization»
- **API** — Memory нет, разработчик сам организует, что подмешивать

### Заметки можно
- **Включить/выключить** (Settings → Memory)
- **Просмотреть** список всех хранимых заметок
- **Удалить** отдельные заметки или очистить всё разом
- **Запросить запоминание вручную** — фразой «запомни, что...»
- **Запросить забывание** — фразой «забудь о...»

### Что важно для приватности
- Заметки хранятся на сервере провайдера (OpenAI, Anthropic и т. д.)
- Провайдер технически имеет к ним доступ
- В корпоративных тарифах политики могут отличаться (заметки не используются для тренировки)
- Подробнее — в Части III

### Чего Memory НЕ делает
- **Не обучает модель.** Веса модели остаются прежними
- **Не персонализирует ответы между всеми пользователями** — заметки только для вашего аккаунта
- **Не передаёт данные между чатами автоматически** — переносятся только заметки, не история чата
- **Не работает между провайдерами** — Memory в ChatGPT не виден Claude и наоборот

## 6. На что обратить внимание

- **Главное слово слайда — «продукт».** Не «приложение», не «сервис», не «программа». «Продукт» — он же подмешивает.
- **Не путать Memory с обучением.** Это разные вещи: Memory подмешивает в промт, обучение — изменяет веса модели. Про обучение — отдельно на слайде 36.
- **Не уходить в детали конкретных продуктов.** Не нужно перечислять, чем Memory в ChatGPT отличается от Projects в Claude. Это про принцип.
- **Голосом подчеркнуть «не модель — продукт».** Это ось всего слайда.
- **Не нагнетать про приватность.** Здесь мы про устройство; про приватность — Часть III. Если зал тревожится — короткий ответ «разберём отдельно».

## 8. Возможные вопросы из зала

**«Так Memory тренирует модель на мне?»**
> «Нет. Это разные вещи. Memory — это хранилище заметок на стороне продукта, которые подмешиваются в правила нового чата. Веса модели при этом не меняются. „Обучение на пользовательских данных“ — отдельный процесс, и о нём поговорим в Части III.»

**«А где эти заметки физически хранятся?»**
> «На серверах провайдера: OpenAI, Anthropic, Google. В базе данных продукта, привязанной к вашему аккаунту. Это не часть модели.»

**«Провайдер их читает?»**
> «Технически — да, доступ у них есть. Юридически — по условиям сервиса. Это вопрос приватности, разбираем в Части III.»

**«Memory есть везде?»**
> «Нет. У разных продуктов разные реализации, у некоторых нет вовсе. В API режиме Memory нет — разработчик сам решает, что подмешивать. У ChatGPT — Memory. У Claude.ai — Projects и Memory. У Gemini — Context personalization.»

**«А можно ли передать Memory в другой аккаунт или другому провайдеру?»**
> «Обычно нет. Это инфраструктура конкретного продукта. Между провайдерами Memory не переносится.»

**«Что значит „можно выключить“? Полностью?»**
> «Да. В настройках есть переключатель — Memory можно отключить совсем. Тогда заметки не накапливаются, и каждый новый чат — действительно с чистого листа. Можно также удалить уже накопленные заметки.»

**«А если я сказал „запомни, что я работаю в ВНИИЛМ“ — это в Memory попадёт?»**
> «Да, обычно — попадёт. Это явный сигнал продукту: „запомни“. В Memory появится заметка вроде „пользователь работает в ВНИИЛМ“.»
