# CONTEXT — глоссарий

Канонический язык проекта. Только термины и их значения — без деталей реализации.

## Деплой

- **svc-what-is-llm** — имя сервиса (Traefik-роутера) в Dokploy, под которым
  публикуются слайды. Это **не** отдельный репозиторий: деплой-конфиг живёт прямо
  в репозитории `seminar-ai-llm`. Ср. с каталогом `svc-*`, где каждый сервис —
  отдельный репозиторий-болванка.

- **blueprint (болванка)** — машинно-независимый репозиторий сервиса (`svc-static`
  и подобные): описывает *что* за сервис (образ, порт контейнера, какие env он
  потребляет, лимиты, сети), но **ничего** привязанного к конкретной машине.

- **two-layer model (двухслойная модель)** — разделение: *слой 1* (git/blueprint)
  описывает сервис машинно-независимо; *слой 2* (только Dokploy, никогда не git) —
  где и как он крутится: реальный домен, секреты, тюнинг лимитов. Шов между слоями
  — ссылки `${VAR}` в `docker-compose.yml`, значения которых даёт Dokploy →
  Environment.

- **SERVICE_DOMAIN** — хост, на который отвечает сервис (например
  `llm.heliotik.xyz`). Задаётся в Dokploy Environment, не в git.

- **SERVICE_NAME** — уникальный id Traefik-роутера на машине (буквы/цифры/`-`, без
  точек и пробелов; для этого сервиса — `llm`). Задаётся в Dokploy Environment.

- **velvetnet-1** — целевой хост: bare-metal Ubuntu 24.04, Dokploy v0.29.4 на
  Docker Swarm, Traefik + Let's Encrypt, wildcard `*.heliotik.xyz`. Публичны
  только 80/443; host-порты контейнеров публиковать нельзя — роутинг только через
  Traefik по сети `dokploy-network`.
