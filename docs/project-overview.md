# Семинар «AI и LLM для исследователей» — обзор проекта

> Этот файл — краткий, но полный паспорт проекта для внешнего ассистента (Claude.ai), который не работал в репозитории. Он позволяет за один прочтённый файл понять, что это, чем сделано, как устроено и в каком оно состоянии.

**Дата семинара:** 13 мая 2026
**Место:** ФБУ ВНИИЛМ
**Ведущие:** Иван Хильков, Андрей Гиценко
**Аудитория:** научные сотрудники, новички в AI/LLM
**Длительность:** 5 ч 15 мин (3 секции по 90 мин + перерывы)

**Главная идея:** объясняем устройство LLM через простые аналогии (Т9 → Smart Compose → GPT → ChatGPT → думающие модели). Понимание устройства само снимает преждевременные страхи — без прямого обращения к теме страхов.

**Главный тезис Секции 1 (слайд 29 ★, keystone):** «Это не интеллект. Это статистика.»

**Источник истины для содержания слайдов:** `stuff/md/section-1-map.md` для Секции 1, `stuff/md/Section_2_slides_draft.md` для Секции 2. Реестр в коде синхронизируется с этими файлами; при расхождениях верят документам.

## 1. Что это технически

Веб-приложение для разработки и показа слайдов:

- **Стек:** Vite 5 + React 18 + React Router v6 + react-markdown (remark-gfm), JavaScript (.jsx), без TypeScript.
- **Менеджер пакетов:** pnpm 10 через mise (Node 25).
- **Слайды:** React-компоненты. Каждый слайд рендерится в фикс-канваса 1920×1080 с динамическим масштабированием под viewport через `useFit`.
- **Заметки ведущего:** Markdown-файлы, рендерятся через `react-markdown` со стилизованным prose.
- **Печать в PDF:** `Cmd+P` → Save as PDF → Landscape; в CSS зашиты `@page { size: 1920px 1080px }` и `@media print` правила.
- **Dev:** `pnpm dev` на `http://localhost:5173/`.
- **Prod:** `pnpm build` → статика в `dist/`.

## 4. Архитектура слайд-движка

### 4.1 Stage + useFit

`<Stage label="07" className="...">{children}</Stage>` рендерит:

```html
<div class="stage">                  /* fixed, top:0 right:0 bottom:0 left:240px (под sidebar) */
  <article class="slide ...">         /* 1920×1080, transform: scale(...) */
    {children}
  </article>
</div>
```

`useFit(enabled)` считает `min(viewportW/1920, viewportH/1080)` (с учётом сайдбара 240px) и применяет `transform: scale(s)` к `.slide`. При ресайзе пересчитывает.

### 4.2 ThumbnailContext (миниатюры)

В `SlideSidebar` тот же `<SlideComponent />` рендерится внутри `<ThumbnailProvider value={true}>`. Внутри:
- `Stage` пропускает обёртку `.stage`, рендерит только `.slide`.
- `useFit` ничего не делает (enabled=false).
- Родительский `.thumb-row__scale` задаёт `transform: scale(0.10416)` (200/1920) → миниатюра 200×112.

Так миниатюры — это **настоящий рендер слайда в маленьком масштабе**, не отдельный код. Работает и для готовых компонентов, и для `StubSlide`-заглушек.

### 4.3 SlideNav

Плавающий бар внизу страницы `/slide/:id`:
- `‹` prev / `X / 101` / `›` next — навигация по `allSlides`
- `↻` — назад на карту: `/opening` (01–05), `/section/1` (06–58), `/section/2` (59–101)
- Клавиатура: `←`/`PageUp`, `→`/`PageDown`/`Space`, `Esc` → карта.

### 4.4 Print (PDF)

В `slide.css`:

```css
@page { size: 1920px 1080px; margin: 0; }

@media print {
  html, body { width: 1920px !important; height: 1080px !important; ... }
  .stage { position: static; left: 0; ... }
  .slide { transform: none !important; page-break-after: always; }
  .slide-sidebar, .slide-nav { display: none !important; }
}
```

Печать `/slide/:id` → один PDF-файл 1920×1080, sidebar и navbar спрятаны.

## 6. Реестр слайдов — контракт

`src/slides/section-1/index.js` (и аналогично `opening/`, `section-2/`):

```js
const ready = (id, type, title, subblock, component, extra = {}) => ({
  id,            // "07"
  type,          // "A" | "B" | "C" | "D" | "E"
  title,         // короткий лейбл для карты (не для самого слайда)
  subblock,      // "1.1 Т9 на Nokia"
  component,     // React-компонент слайда
  done: true,    // ★ готов или нет — отдельно от наличия компонента
  hasNotes: extra.hasNotes !== false,   // есть ли .md в src/notes/section-1/
  keystone: !!extra.keystone,           // главный тезис (29) — акцентная обводка
});

const draft = (id, type, title, subblock, extra = {}) => ({
  id, type, title, subblock,
  component: null,    // → отрендерится через StubSlide
  done: false,        // ★ карта показывает как черновик
  hasNotes: false,
  keystone: !!extra.keystone,
});
```

**Важное разделение:**
- `component` — есть ли React-компонент для рендера (если нет → StubSlide).
- `done` — считаем ли слайд готовым (карта читает это поле; черновики в карте — `is-draft`).

Все слайды Секции 1 (53) и Секции 2 (43) зарегистрированы. Черновики получают `StubSlide` рендер автоматически.

`src/slides/registry.js` объединяет три реестра:

```js
export const allSlides = [...openingSlides, ...section1Slides, ...section2Slides];   // 5 + 53 + 43 = 101
export function findSlide(id);    // null если не существует
export function sectionOf(id);    // 'opening' | 'section-1' | 'section-2' | null
```

## 8. Карты секций

**`/section/1`** и **`/section/2`** — общие стили (`Section1.css`), общий React-шаблон (карточка / part-grid / легенда). Различаются только данными из реестра и параметром `PART_ORDER`.

Клик по карточке → `/slide/:id`. Клик по «✎» → `/notes/:id` (stopPropagation).

## 9. Speaker notes

`.md` файлы в `src/notes/`. Vite собирает их через `import.meta.glob('*.md', { query: '?raw', eager: true })` — никакого рантайм-фетча.

**Структура заметки** (см. `_template.md`):

1. Заголовок слайда + блок-метаданные (тип, хронометраж, ведущий, цель)
2. Технический фон (для уверенности ведущего)
3. Концептуальный мост (как связан с другими слайдами)
4. Полу-сценарий выступления (прямая речь в blockquotes)
5. На что обращать внимание (темп, паузы, реакция зала)
6. Возможные вопросы (Q&A)
7. Связки со слайдами

**Стилизация (`src/styles/prose.css`):**
- IBM Plex Sans, 17px / 1.6 line-height
- Первый blockquote (метаданные) — выделен серым фоном `--hi`
- Остальные blockquotes — левый border accent (`--accent`)
- Таблицы с моно-заголовками, светлой подложкой
- Inline code и pre с моно-фоном

**Покрытие заметками** (на момент написания): открытие 01–03, Секция 1: 07, 08, 09, 10, 11, 12.

### 10.4 Tone of voice (контент)

- Без снисходительности и патернализма.

- Через метафоры, не через жаргон.

- Главные метафоры: Т9 → автозаполнение → GPT (лестница угадывания), LLM как «попугай, проглотивший интернет», ChatGPT как «дрессированный начитанный человек» (RLHF), думающие модели как «студент с черновиком».

## 11. Workflow разработки

### Запуск (с нуля на новой машине)

```bash
mise install        # подтягивает Node 25 + pnpm последний по .mise.toml
pnpm install
pnpm dev            # http://localhost:5173/
```

### Создать новый слайд (например, 14 — Transformer)

1. Создать `src/slides/section-1/Slide14.jsx`. Скопировать ближайший по типу:
   - **C-визуал** → `Slide07.jsx` (для SVG-устройства), `Slide08.jsx` (для схемы), `Slide11.jsx` (для UI-mock), `Slide12.jsx` (для UI-mock с подсказками)
   - **B-тезис** → шаблон ещё не выделен, можно собрать вручную в стиле `Slide09` (центральная фраза)
   - **D-сравнение** → скопировать `Slide10.jsx` + `Slide10.css`
   - **E-момент** → собрать через `<ESlide variant="momentum|number|live" ... />`
   - **A-дивайдер** → `<ASlide num="..." roman="..." meta="..." title="..." />`
2. В соответствующем `index.js` (`section-1` или `section-2`):
   - Импортировать компонент.
   - Заменить строку `draft('XX', ...)` на `ready('XX', 'TYPE', 'Title', 'Подблок', SlideXX[, { hasNotes: false }])`.
3. Слайд автоматически появится:
   - По адресу `/slide/XX` — настоящий рендер вместо stub.
   - В карте секции — карточка с тинтом фона.
   - В sidebar `/slide/:id` — миниатюра с настоящим рендером.

### Добавить speaker notes

```bash
cp src/notes/section-1/_template.md src/notes/section-1/slide-XX.md
# отредактировать
```

В реестре убедиться, что нет `{ hasNotes: false }` (по умолчанию true). Vite через `import.meta.glob` подхватит автоматически.

### Печать в PDF

Открыть `/slide/XX` → `Cmd+P`/`Ctrl+P` → Save as PDF → Layout: Landscape → Margins: None → Background graphics: ON → Save. Получается один файл 1920×1080.

### Прод-сборка

```bash
pnpm build      # → dist/
pnpm preview    # локальный smoke
```

## 16. Контракт для ассистента-помощника

Если вы (Claude.ai) хотите помочь:

- **Добавить слайд С1**: текст и тип брать из `stuff/md/section-1-map.md`. Если в `Section 1 Skeleton.html` есть подходящий текст с тем же смыслом — переиспользовать, но **не** сверяться по нумерации Skeleton (она устарела).
