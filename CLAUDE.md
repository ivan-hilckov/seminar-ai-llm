# Семинар «AI и LLM для исследователей»

Проект подготовки слайдов для однодневного семинара в ФБУ ВНИИЛМ.

**Дата проведения:** 13 мая 2026
**Ведущие:** Иван Хильков · Андрей Гиценко
**Аудитория:** научные сотрудники, новички в AI
**Длительность:** 5 часов 15 минут

## Концепция

Объясняем устройство и принципы работы больших языковых моделей через простые аналогии. Понимание устройства снимает преждевременные опасения само — без прямого обращения к теме страхов.

## Визуальная система

Стили зафиксированы в `src/styles/` (`global.css` — палитра и типографика, `slide.css` — сцена и печать). Слайды используют CSS-классы оттуда; компонентные стили лежат рядом с компонентом (`SlideXX.css`).

**Палитра:**
- `--bg: #FAFAF7` — фон (off-white)
- `--text: #1A1A1A` — основной текст (near-black)
- `--text-secondary: #6B6B68` — серый
- `--line: #D9D7CF` — линии и рамки
- `--accent: #8B2A1F` — тёплый красный, ТОЛЬКО на главных moment-слайдах
- `--highlight: #F0EEE8` — мягкая подсветка в схемах

**Типографика:** IBM Plex Sans (основной), IBM Plex Mono (мета).

**Размер слайда:** 1920×1080 (16:9). На экране масштабируется через transform: scale(0.667).

| Тип | Назначение | Композиция |

| **A** | Section divider | Огромная римская цифра, название части, мета |

## Принципы подачи

**Tone of voice:**
- Без снисходительности и патернализма
- Без позиционирования через страх
- Через метафоры, не через жаргон
- Тезисы конкретные, инфостиль

**Запретные слова в формулировках:** «страх», «бояться», «не страшно», «поручить», «мифы».

**Метафоры:**
- Т9 → автозаполнение → GPT — «лестница угадывания»
- LLM как «попугай, проглотивший интернет»
- ChatGPT как «дрессированный начитанный человек» (RLHF)
- Думающие модели как «студент с черновиком»
- Главный тезис Секции 1: **«Это не интеллект. Это статистика.»**

## Стек

Vite + React 18 + React Router v6 + react-markdown (remark-gfm).
JavaScript (.jsx), без TypeScript. Dev на `http://localhost:5173/`.

## Workflow

### Запуск dev-сервера

```bash
mise install         # один раз — node + pnpm
pnpm install         # один раз — зависимости
pnpm dev             # vite на :5173
```

### Создать новый слайд (например, 11)

1. Создать `src/slides/section-1/Slide11.jsx`. Скопировать ближайший по типу:
   - **C-визуал:** взять `Slide07.jsx` или `Slide08.jsx`
   - **B-тезис:** проще всего собрать в стиле `Slide09.jsx` (одна большая фраза по центру)
   - **D-сравнение:** скопировать `Slide10.jsx` + `Slide10.css`
   - **E-момент:** скопировать `Slide09.jsx`
2. Изменить контент + `meta` (id/type/title/subblock).
3. В `src/slides/section-1/index.js`: импортировать компонент и заменить строку `draft('11', ...)` на `ready('11', ...)`.
4. Слайд автоматически появится по адресу `/slide/11` и в карте `/section/1`.

### Speaker notes

Файлы в `src/notes/section-1/slide-XX.md`. Создаются копированием `_template.md`. Vite собирает их статически (`import.meta.glob`) — после добавления нового .md перезапустить dev-сервер не нужно, HMR подхватит.

### Печать в PDF

Открыть `/slide/XX`, Cmd+P (Ctrl+P) → Save as PDF → Layout: Landscape. В `slide.css` зашиты `@page { size: 1920px 1080px }` и `@media print` правила.

### Прод-сборка

```bash
pnpm build           # → dist/
pnpm preview         # локальный preview сборки
```

## Реестр слайдов (модель данных)

Каждая секция держит свой реестр-массив в `index.js` и собирает его через фабрики:

- `ready(id, type, title, subblock, component, extra = {})` → `{ ..., component, done: true, hasNotes, keystone }`. `hasNotes` по умолчанию `true` (есть `.md` в `src/notes/`), отключается через `{ hasNotes: false }`.
- `draft(id, type, title, subblock, extra = {})` → `{ ..., component: null, done: false, hasNotes: false }`. Слайд-черновик рендерится заглушкой `StubSlide` (пунктирный скелет).
- `extra.keystone: true` помечает кульминационный тезис акцентной обводкой (слайд 29).

`src/slides/registry.js` объединяет реестры всех секций (`opening` + `section-1` + `section-2`) и даёт единый доступ:

- `findSlide(id)` — запись по id из любой секции или `null`
- `allSlides` — полный плоский список (используется на `/` и в навигации)
- `sectionOf(id)` → `'opening' | 'section-1' | 'section-2' | null`

Роуты (`src/App.jsx`): `/`, `/opening`, `/section/1`, `/section/2`, `/slide/:id`, `/notes/:id`, `/print`, `*`.

## Неочевидное

- **Дублирование `meta`.** Каждый `SlideXX.jsx` экспортирует свой локальный `meta`, повторяющий поля реестра (`id`/`type`/`title`). Эти два источника могут разойтись (напр., у Slide07 в реестре `title: 'Помните это?'`, а в файле `meta.title: 'С этого всё началось'`). Источник истины для карты секции — реестр `index.js`, а не локальный `meta`. При правке заголовка меняйте оба.
- **Заметки грузятся на сборке.** `src/routes/Notes.jsx` тянет `.md` через `import.meta.glob([...], { query: '?raw', import: 'default', eager: true })` — никаких рантайм-fetch. Новый `.md` подхватывается HMR без перезапуска dev-сервера, но имя должно быть `slide-XX.md` (id с ведущим нулём).
- **Тестов, линтера, форматтера и typecheck нет.** В `package.json` только `dev`/`build`/`preview`. Не ищите `pnpm test`/`pnpm lint` — их нет по дизайну.
- **Источник истины по контенту слайдов** — `stuff/md/section-1-map.md` (Секция 1), доступен через symlink `public/stuff/`. Отдельного исходника-драфта по Секции 2 в репозитории пока нет.
