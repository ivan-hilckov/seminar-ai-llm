## Визуал

D-шаблон. Без иллюстраций. Чистая типографика.

### Структура

Сверху вниз:
1. Мета-тег подблока (моно, `--mute`, заглавными)
2. Заголовок слайда («Что меняется»)
3. Две колонки, разделённые вертикальной 1px-линией `--rule`
4. Горизонтальная 1px-линия и summary внизу

### Внутри каждой колонки

Сверху вниз:
1. Подзаголовок колонки крупно (IBM Plex Sans, `--ink`)
2. Моно-подпись (IBM Plex Mono, `--mute`, мельче)
3. Воздух
4. 5 строк-примеров (IBM Plex Sans, `--ink`, средний размер), с равным воздухом между строками

### Главный визуальный приём

Параллельность колонок. Глаз должен «сканировать сверху вниз» обе колонки и видеть, что на каждой высоте противопоставление «короткое / сложное». Списки одной длины, одного формата, одинакового размера шрифта.

## Композиция слайда

Стандартный D-шаблон с summary:
- Заголовок сверху
- Две равные колонки с вертикальной 1px-линией между ними
- Summary внизу через горизонтальную 1px-линию

## Чего избегать
- НЕ добавлять иконки, чек-маркеры ✓, плюсики/минусики, пунктирные буллеты — это снижает плотность
- НЕ нумеровать строки в колонках
- НЕ использовать бордовый `--accent` — зарезервирован за слайдом 32
- НЕ делать одну колонку ярче / больше / в рамке — обе колонки равноправны
- НЕ менять формулировки задач — это согласованные примеры из контекста аудитории (научные сотрудники ВНИИЛМ)
- НЕ растягивать summary в две строки — это одна короткая строка-вывод
- НЕ переименовывать summary в «Вывод», «Главное» — просто строка по центру без префикса
