## Визуал

B-слайд без anchor. Никаких иллюстраций, схем или иконок. Композиция строится только на типографике:

- Сверху — мета-тег подблока (служебный, моноширинный, мелкий, серый `--mute`)
- Под ним — заголовок (средний кегль, `--ink`)
- По центру — тезис из двух предложений (крупный кегль, 44–56px, `--ink`)
- Снизу — пусто (anchor отключён)

Главный визуальный приём — парность тезиса. Два предложения зеркальны по структуре: «Полезна для X. Мешает для Y.». Они должны читаться как противопоставление, а не как два отдельных тезиса.

### Размещение тезиса
Два предложения — на двух строках, выровнены по левому краю относительно центрального оптического блока. «Полезна для предпочтений.» — первая строка. «Мешает для фактов.» — вторая строка.

Точка после каждого предложения — обязательна (диктует ритм паузы).

## Что переиспользовать
- Универсальный `BSlide` или композицию из готового B-слайда без anchor

## Чего избегать
- Иконок (замочек, мозг, лампочка) — никаких пиктограмм
- Колоночной вёрстки «полезна / мешает» — это B, не D. Антитезис в самом тексте, не в раскладке
- Использования `--accent` для подсветки слов «Полезна» / «Мешает»
- Bullet-list или enumeration
- Подзаголовков под тезисом
