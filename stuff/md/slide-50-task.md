## Визуал

Два визуально симметричных SVG-«окна ChatGPT» — рамка с тонкой линией `--rule`, скругление 6–8px, фон `--bg` или чуть светлее, внутри текст моноширинным жанром (но не монo-шрифтом — IBM Plex Sans, как везде). Цель — узнаваемость интерфейса без буквального копирования OpenAI-брендинга.

Главный фокус — на содержимом «окон». Шрифт внутри окна — стандартный body, чуть мельче основного текста слайда. Выдуманные реквизиты выделяются **волнистым подчёркиванием `--accent`** (единственное использование акцентного цвета за пределами keystone-слайда 32).

Параллелизм колонок: одинаковая высота рамок, одинаковая структура (рамка → текст → подпись под рамкой). Это важно для эффекта «оба ответа выглядят одинаково правдоподобно».

## Композиция слайда

D-шаблон:
- Заголовок сверху + подзаголовок мельче, серым
- Две колонки с тонкой 1px-линией `--rule` между ними
- В каждой колонке: заголовок колонки → SVG-«окно» с примером → подпись под окном (`--mute`)
- Summary одной строкой внизу по центру

Без мета-метки подблока сверху.

Волнистое подчёркивание — тонкое, не отвлекает от чтения. Подчёркиваются только выдуманные реквизиты (название дела + цитата суда; шифр ГОСТа + название документа). Не подчёркивается весь текст ответа.

## Чего избегать
- Не копировать буквально интерфейс ChatGPT (логотип, точные цвета OpenAI) — это стилизация, не репродукция
- Не выделять выдуманное красным фоном/обводкой/иконкой ⚠️ — только тонкое волнистое подчёркивание
- Не подчёркивать весь текст ответа — только конкретные выдуманные реквизиты
- Не вставлять смайлики, иконки «ошибка», восклицательные знаки — слайд работает на контрасте «выглядит правдоподобно / не существует»
- Не делать левую колонку визуально «тревожнее» правой — обе одинаково спокойные
