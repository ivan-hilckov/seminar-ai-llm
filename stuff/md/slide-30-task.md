## Визуал

D-шаблон с акцентом на цифры. Без иллюстраций.

### Структура колонки сверху вниз

1. Имя модели крупно (DeepSeek R1 / DeepSeek V3) — как подзаголовки в Slide28
2. Моно-подпись режима (`думающая модель` / `обычная модель`)
3. Воздух
4. **Огромное число с символом %** (14.3% / 3.9%) — главный визуальный элемент
5. Подпись «галлюцинаций» под числом

### Структура слайда сверху вниз

1. Мета-тег подблока (моно, `--mute`, заглавными)
2. Заголовок слайда («У думающих моделей выше галлюцинации»)
3. Две колонки, разделённые вертикальной 1px-линией `--rule`
4. Горизонтальная 1px-линия
5. Summary по центру
6. Источник внизу мелким моно

### Главный визуальный приём

Контраст 14.3% vs 3.9%. Оба числа:
- На одной горизонтали
- Одного размера
- Одного цвета (`--ink`)
- По центру своей колонки

Никакого подсвечивания «плохой» цифры красным. Контраст создаётся самим размером 14.3 vs 3.9 — это в 3.7 раз. Глаз должен это считать без подсказки.

## Композиция слайда

Стандартный D-шаблон с summary + источник:
- Заголовок сверху
- Две равные колонки с вертикальной 1px-линией
- Summary внизу через горизонтальную 1px-линию
- Источник — мелким моно

## Реализация в проекте

Откуда копировать структуру: `Slide28.jsx` или `Slide29.jsx` (ближайшие D-шаблоны section-1)

### Регистрация
В `src/slides/section-1/index.js` заменить:
- было: `draft('30', 'D', 'Ловушка — выше галлюцинации', '1.5')`
- стало: `ready('30', 'D', 'У думающих моделей выше галлюцинации', '1.5', Slide30)`

> Обратить внимание: заголовок обновляется с «Ловушка — выше галлюцинации» (метафорический) на «У думающих моделей выше галлюцинации» (фактический).

### Реализация
- Use CSS variables (`--ink`, `--mute`, `--mute-2`, `--rule`)
- Шрифты — IBM Plex Sans (контент + крупные числа), IBM Plex Mono (мета-тег, подписи режимов, источник)
- Числа могут переиспользовать стиль из ESlide (там есть готовое крупное число)

## Что переиспользовать
- D-шаблон из `Slide28.jsx` или `Slide29.jsx`
- Стиль крупного числа из `ESlide` (если в проекте уже отрендерено E-число типа «175 миллиардов параметров»)

## Чего избегать
- НЕ красить число 14.3% в `--accent` или красный — бордовый зарезервирован за 32. Контраст создаётся размером цифры, а не цветом
- НЕ добавлять стрелочки ↑↓, восклицательные знаки, иконки опасности
- НЕ делать одну колонку ярче или больше — обе равноправны, цифры говорят сами
- НЕ переходить на эмоциональный язык в подписях («плохо», «опасно», «осторожно»)
- НЕ растягивать source-line — это короткая подпись, не описание методики
- НЕ переименовывать «галлюцинаций» во «вранья», «выдумок», «ошибок» — термин уже принят в индустрии, его и закрепляем для слайдов 51–54
- НЕ дробить summary в две строки — одна короткая строка-вывод
