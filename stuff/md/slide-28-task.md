## Визуал

D-шаблон. Без иллюстраций. Чистое типографическое сравнение.

### Структура

Сверху вниз:
1. Мета-тег подблока (моно, --mute, заглавными)
2. Заголовок слайда («Обычная модель vs думающая»)
3. Две колонки, разделённые вертикальной 1px-линией `--rule`
4. Опциональная горизонтальная разделительная линия и summary внизу

### Внутри каждой колонки

Сверху вниз:
1. Подзаголовок колонки крупно («Студент-выскочка» / «Студент с черновиком») — IBM Plex Sans, --ink, размер примерно как у других D-слайдов
2. Три блока, каждый с лейблом и формулировкой:

```
ПОВЕДЕНИЕ              (моно, --mute, мелкий)
Отвечает сразу         (sans, --ink, средний)

ХАРАКТЕР
Быстрый, без проработки

НАЗНАЧЕНИЕ
Для коротких вопросов
```

Лейблы (Поведение / Характер / Назначение) выровнены параллельно в обеих колонках — глаз должен «сканировать горизонтально» три пары.

### Summary внизу

Одна строка по центру, под обеими колонками. Шрифт — IBM Plex Sans, размер меньше заголовков колонок, цвет --ink. Опционально сверху над summary — тонкая 1px-линия --rule на всю ширину контентной области.

### Главный визуальный приём

Три параллельные оси сравнения. Лейблы слева в обеих колонках идут на одной высоте — это даёт глазу ритм и подчёркивает, что мы сравниваем по одинаковым критериям.

## Композиция слайда

Стандартный D-шаблон с opt-summary:
- Заголовок сверху
- Две колонки, разделённые вертикальной 1px-линией
- Summary внизу через горизонтальную 1px-линию

## Реализация в проекте

### Регистрация

> Обратить внимание: заголовок в `index.js` обновляется с «Студент-выскочка vs студент с черновиком» (метафорический) на «Обычная модель vs думающая» (технический). Метафорические заголовки уходят в подзаголовки колонок.

### Реализация
- Чисто типографика, без SVG-иллюстраций
- Use CSS variables (`--ink`, `--mute`, `--rule`)
- Шрифты — IBM Plex Sans (контент), IBM Plex Mono (лейблы и мета-тег) из `src/styles/global.css`

## Что переиспользовать
- D-шаблон из `Slide10.jsx` (структура двух колонок с 1px-линией)
- Стили лейблов из любого готового слайда (если есть моно-метки)

## Чего избегать
- НЕ добавлять иконки студентов, лампочек, голов, мозгов
- НЕ использовать бордовый `--accent` — зарезервирован за слайдом 32
- НЕ делать одну колонку ярче / больше / в рамке — обе колонки равноправны (это не «правильно vs неправильно», это «разные режимы»)
- НЕ ставить стрелку, плюсики/минусы, эмодзи галочки/крестика между колонками
- НЕ добавлять оценочных эпитетов («умный», «глупый», «лучше», «хуже»)
- НЕ переводить «выскочка» в положительное русло — он не плохой, просто другой
- НЕ растягивать summary в две строки — это одна короткая строка-вывод
