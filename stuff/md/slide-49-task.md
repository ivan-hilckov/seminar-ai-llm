# Слайд 49 · Не знает фактов — знает статистику текстов

## Метаданные
- Тип: B (тезис)
- Подблок: 3.4 Точность и галлюцинации
- Хронометраж: 1.5 минуты
- Ведущий: Иван

## Контент слайда

### Заголовок
Не знает фактов — знает статистику текстов

### Тезис (центральный, крупно)
Модель не хранит факты — она восстанавливает их по статистике текстов

### Anchor (снизу, мельче, серым)
Пустоты в знании заполняются тем, что статистически похоже на правду

### Мета-тег подблока
Нет (осознанно убран на этом слайде)

## Визуал

Слайд без иллюстрации — чистый B-шаблон с тремя текстовыми зонами:
- заголовок сверху
- тезис по центру
- anchor снизу

Главный фокус — на тезисе по центру. Слайд не нагружен графикой — это смысловой узел перед примерами галлюцинаций на слайде 50.

## Композиция слайда

Стандартный B-шаблон без мета-метки подблока:

- Заголовок сверху (стандартный размер заголовка для B), без мета-тега над ним
- Тезис по центру слайда, 44–56px, тёмный (`--ink`)
- Anchor снизу — одна строка, размер мельче (≈ 24–28px), цвет `--mute`
- Между тезисом и anchor — достаточно воздуха, anchor не «прижимается» к нижнему краю

Никакого `--accent` (бордового). Никаких иконок, линий-разделителей, рамок.

## Что переиспользовать
- Стандартный B-каркас (заголовок + центральный тезис + anchor) из ближайшего готового B-слайда
- Шрифтовая шкала B-типа из `global.css` / общих стилей
- Цветовые токены: `--ink` для тезиса, `--mute` для anchor

## Чего избегать
- Не добавлять иконку «галлюцинация», «факт», «вопрос» — слайд намеренно текстовый
- Не выделять цветом отдельные слова в тезисе (никаких подчёркиваний волной, никакого `--accent`)
- Не превращать anchor в bullet-list — это одна строка пояснения, не перечисление
- Не делать тезис на две одинаковые строки — оптический баланс важнее ровной разбивки
