# Слайд 56 · Главные тезисы

## Метаданные
- Тип: B (тезисный, расширенный — список из 5 равноправных фраз)
- Подблок: Закрытие Секции 1 (финальный слайд секции)
- Хронометраж: 1.5 минуты
- Ведущий: Иван

## Контент слайда

### Заголовок
Главные тезисы

### Пять тезисов (один под другим, без нумерации, без маркеров)

1. Это не интеллект — статистика
2. Без контекста модель не видит вашу задачу
3. Каждый ответ модель собирает заново
4. Проверяй факты, цифры, ссылки
5. AI берёт на себя задачи, не роль

(нумерация в этом задании — для удобства Claude Code, на слайде её нет)

### Мета-тег подблока сверху
Нет (единство финальной трети Секции 1)

## Визуал

Без иллюстрации. Чистая типографическая композиция.

Структура:
- Заголовок сверху
- Пять тезисов один под другим, оптически по центру слайда
- Каждый тезис — отдельная строка, средне-крупный размер
- Все тезисы одного веса и цвета `--ink`, без иерархии
- Без нумерации, без маркеров-кружков, без буллетов, без дефисов в начале строки
- Между тезисами достаточный воздух — каждый читается как отдельная мысль, не как пункт списка

Размеры (ориентир):
- Заголовок слайда — стандартный для B
- Тезисы — 32–40px, `--ink`, обычный/средний вес
- Между тезисами — увеличенный вертикальный отступ (1.2–1.5× от высоты строки), чтобы фразы «дышали»

Тире в первом тезисе («Это не интеллект — статистика») — типографское `—` с пробелами. Запятая в пятом («задачи, не роль») — обычная.

Без `--accent`, без иконок, без галочек/крестиков. Слайд работает на простоте: пять фраз и больше ничего.

## Композиция слайда

Слайд занимает центр канваса:
- Заголовок прижат к верхней рабочей линии
- Пять тезисов распределены по вертикали, оптически центрированы
- Левое выравнивание текста по единой левой границе (все строки начинаются с одной вертикали)

Длинная строка («Без контекста модель не видит вашу задачу») и короткая («Это не интеллект — статистика») должны выглядеть гармонично — за счёт равных вертикальных интервалов и единого размера шрифта, не за счёт выравнивания по правому краю.

## Реализация в проекте

### Компонент
Где создать: `src/slides/section-1/Slide56.jsx`
Откуда копировать структуру: ближайший B-слайд с центральным текстовым блоком. Возможно — Slide53 (если готов): там тоже несколько строк одного веса в центре. Адаптировать под 5 строк.

### Регистрация
В `src/slides/section-1/index.js` заменить:
- было: `draft('56', 'E', 'Главные тезисы Секции 1', 'Закрытие'),`
- стало: `ready('56', 'B', 'Главные тезисы', 'Закрытие', Slide56),`

Внимание: меняется тип (E → B — в карте было E, фактически слайд работает как B с расширенным блоком) и заголовок (короче, без «Секции 1»).

## Что переиспользовать
- B-каркас (заголовок + центральный текстовый блок)
- Цветовые токены: `--ink` для всех пяти тезисов
- Шрифтовая шкала из готовых B-слайдов с центральным блоком (например Slide53 если готов)

## Чего избегать
- Не нумеровать тезисы (1, 2, 3... или I, II, III...). Слайд не «чек-лист», а смысловой ландшафт.
- Не добавлять маркеры-кружки слева (это формат слайда 55, а 56 — другой регистр).
- Не использовать `--accent` ни на одном тезисе, включая первый (повтор keystone). Здесь keystone — равноправный пункт, выделение нарушит идею «пять смысловых якорей».
- Не выделять никакой тезис жирным/курсивом/цветом. Иерархия не нужна.
- Не разносить тезисы по двум колонкам. Только вертикальная композиция.
- Не добавлять подзаголовок («Что выносим из Секции 1» или подобное) — минималистично.
- Не растягивать слайд анимацией появления тезисов один за другим (анимации запрещены проектом).
